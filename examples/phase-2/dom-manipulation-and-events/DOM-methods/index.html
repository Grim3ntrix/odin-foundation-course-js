Examples

1. Altering elements

// creates a new div referenced in the variable 'div' const div = document.createElement("div");

2. Adding inline style

// adds the indicated style rule to the element in the div variable div.style.color = "blue";


// adds several style rules div.style.cssText = "color: blue; background: white;";


// adds several style rules div.setAttribute("style", "color: blue; background: white;");

Quick Note

When accessing a kebab-cased CSS property like background-color with JS, you will need to either use camelCase with dot notation or bracket notation.

When using bracket notation, you can use either camelCase or kebab-case, but the property name must be a string.

Example

| wrong way |

// dot notation with kebab case: doesn't work as it attempts to subtract color from div.style.background

// equivalent to: div.style.background - color

div.style.background-color;

| correct way |

// dot notation with camelCase: works, accesses the div's background-color style

div.style.backgroundColor;

// bracket notation with kebab-case: also works

div.style["background-color"];

// bracket notation with camelCase: also works

div.style["backgroundColor"];

3. Editing attributes

// if id exists, update it to 'theDiv', else create an id with value "theDiv"

div.setAttribute("id", "theDiv");

// returns value of specified attribute, in this case "theDiv"

div.getAttribute("id");

// removes specified attribute

div.removeAttribute("id");

4. Working with classes

// adds class "new" to your new div

div.classList.add("new");

// removes "new" class from div

div.classList("remove");

// if div doesn't have class "active" then add it, or if it does, then remove it

div.classList.toggle("active");

Note: It is often standard (and cleaner) to toggle a CSS style rather than adding and removing inline CSS.

5. Adding text content

// creates a text node containing "Hello World!" and inserts it in div

div.textContent = "Hello World!";

6. Adding HTML content

// renders the HTML inside div

div.innerHTML = "Hello World!";

Note: Note that using textContent is preferred over innerHTML for adding text, as innerHTML should be used sparingly to avoid potential security risks. To understand the dangers of using innerHTML, watch this video about preventing the most common cross-site scripting attack.
